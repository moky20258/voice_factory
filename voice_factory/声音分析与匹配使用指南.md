# 声音分析与声模匹配建议系统使用指南

## 📋 功能概述

本系统提供完整的声音分析和声模匹配功能：

1. **声音特征分析** - 提取 F0、性别、音色、频谱等特征
2. **预训练声模匹配** - 与6个理论最优声模对比
3. **已训练声模匹配** - 与20个已训练的实际声模对比
4. **智能建议生成** - 提供详细的使用和训练建议

## 🚀 快速开始

### 方式一：命令行快速使用

```bash
# 分析单个音频
python voice_analysis_and_recommendation.py audio.wav

# 分析音频目录
python voice_analysis_and_recommendation.py ./audio_folder

# 保存报告
python voice_analysis_and_recommendation.py audio.wav -o report.json

# 返回更多匹配结果
python voice_analysis_and_recommendation.py audio.wav --top-n 5
```

### 方式二：批处理文件

双击运行：
- `声音分析与匹配.bat` - 命令行版本
- `声音分析与匹配-交互版.bat` - 交互式版本（推荐新手）

### 方式三：交互式界面

```bash
python interactive_voice_matcher.py
```

## 📖 详细使用说明

### 1. 命令行版本

#### 基本用法

```bash
python voice_analysis_and_recommendation.py <音频路径> [选项]
```

#### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `audio_path` | 音频文件路径或目录（必需） | - |
| `-o, --output` | 输出报告文件路径（JSON） | 不保存 |
| `--top-n` | 返回前 N 个匹配结果 | 3 |
| `--rvc-logs` | RVC训练日志目录 | `W:\rvc\logs` |
| `--no-pretrained` | 跳过预训练声模匹配 | False |
| `--no-trained` | 跳过已训练声模匹配 | False |

#### 示例

```bash
# 分析单个文件，不保存报告
python voice_analysis_and_recommendation.py my_voice.wav

# 分析目录，保存报告
python voice_analysis_and_recommendation.py ./my_audios -o my_report.json

# 只匹配预训练声模
python voice_analysis_and_recommendation.py audio.wav --no-trained

# 只匹配已训练声模
python voice_analysis_and_recommendation.py audio.wav --no-pretrained

# 返回10个匹配结果
python voice_analysis_and_recommendation.py audio.wav --top-n 10
```

## 🎯 使用场景

### 场景一：快速找到可用声模

**目标**: 输入声音后，快速找到最匹配的已训练声模


**结果**: 直接获得可使用的声模文件路径

**结果**: 获得理论最优的预训练声模和训练建议

**结果**: 获得目录下所有音频的平均特征和匹配建议

**目标**: 对比理论推荐和实际可用的声模

**结果**: 完整的对比报告，帮助决策是直接使用还是训练新声模

## 💡 匹配度解读

### 预训练声模匹配度

| 匹配度 | 含义 | 建议 |
|--------|------|------|
| 80-100% | 极高匹配 | 使用该预训练模型训练效果最佳 |
| 60-80% | 高匹配 | 可以使用，训练时间可能稍长 |
| 40-60% | 中等匹配 | 可以使用，但需要更多训练 |
| <40% | 低匹配 | 建议考虑其他声模或跨性别训练 |

### 已训练声模匹配度

| 匹配度 | 含义 | 建议 |
|--------|------|------|
| 80-100% | 极高匹配 | 强烈推荐直接使用 |
| 70-80% | 高匹配 | 推荐使用，效果良好 |
| 60-70% | 中等匹配 | 可以使用，效果可接受 |
| <60% | 低匹配 | 建议训练新声模 |

## 🔧 高级配置

### 自定义 RVC 日志目录

如果你的 RVC 训练日志不在默认位置：

```bash
python voice_analysis_and_recommendation.py audio.wav --rvc-logs "D:\MyRVC\logs"
```

### 修改声模数据库

编辑 `voice_models_db.json` 可以：
- 添加新的预训练声模
- 调整匹配权重
- 修改声模特征参数

### 更新已训练声模数据库

删除 `trained_voices_db.json` 后重新运行，系统会自动扫描最新的训练结果。

## 📝 注意事项

1. **音频质量要求**:
   - 建议时长：3-30秒
   - 格式：WAV (40kHz, 16bit)
   - 内容：清晰的说话声，无背景音乐

2. **匹配准确性**:
   - 音频越长，特征提取越准确
   - 建议分析目录而非单个文件（取平均特征）
   - 已训练声模的匹配基于 F0 和 Embedding 相似度

3. **性能优化**:
   - 目录分析默认采样10个文件（可修改代码）
   - 首次扫描已训练声模需要几秒
   - 后续运行会加载缓存数据库

## 🐛 故障排除

### 问题1：ModuleNotFoundError: No module named 'librosa'

**解决**:
```bash
pip install librosa
```

### 问题2：找不到音频文件

**解决**:
- 检查路径是否正确
- 使用绝对路径
- 确保文件存在且格式为 WAV

### 问题3：匹配度都很低

**可能原因**:
- 音频质量差（噪音、剪辑）
- 特殊音色（儿童、老人、特殊发声）
- 性别跨越（男→女或女→男）

**解决**:
- 提高音频质量
- 使用目录分析（多个样本）
- 考虑训练新声模

### 问题4：已训练声模匹配度都是20%

**原因**: 数据库未更新

**解决**:
```bash
# 删除缓存数据库
del trained_voices_db.json

# 重新运行（会自动扫描）
python voice_analysis_and_recommendation.py audio.wav
```

## 🎓 技术原理

### 声音特征提取

1. **基频 (F0)**: 使用 librosa.pyin 提取
2. **频谱质心**: 反映音色亮度
3. **共振峰**: 反映声道特征
4. **MFCC**: 梅尔频率倒谱系数
5. **响度**: RMS 能量

### 匹配算法

1. **预训练声模**: 
   - F0 均值匹配 (40%)
   - F0 范围匹配 (20%)
   - 性别匹配 (15%)
   - 频谱质心匹配 (15%)
   - 音色类型匹配 (10%)

2. **已训练声模**:
   - Embedding 余弦相似度 (50%)
   - F0 高斯相似度 (40%)
   - 基础分 (10%)
