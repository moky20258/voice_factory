# 🎯 RVC 训练最简操作指南

## ⚠️ 重要说明

RVC 的训练功能**深度集成在 WebUI 中**，无法通过纯命令行完全自动化。这是 RVC 的设计限制。

但是！你只需要** 5 步操作**，每次只需 10 秒钟！

---

## 🚀 最简训练流程（每个 speaker 只需 10 秒）

### 第 1 步：启动 RVC（只需做一次）

双击运行：
```
W:\rvc\go-web.bat
```

等待看到：
```
Running on local URL:  http://127.0.0.1:7897
```

### 第 2 步：打开浏览器

访问：http://localhost:7897

### 第 3 步：点击"训练"标签

在页面顶部找到并点击 **"训练"**

### 第 4 步：填写信息（10 秒）

在"实验名称"输入框输入：
```
speaker_0001
```

其他全部**保持默认**！

### 第 5 步：点击"一键训练"

点击页面底部的 **"一键训练"** 按钮

---

## ⏱️ 然后等待

- **GPU**: 30-60 分钟后完成
- **CPU**: 2-4 小时后完成

训练完成后，模型自动保存在：
```
W:\rvc\logs\speaker_0001\G_200.pth       ← 可用模型
W:\rvc\logs\speaker_0001\added_200.index ← 可用索引
```

---

## 📋 批量训练 20 个 Speaker

训练完 speaker_0001 后，重复第 4-5 步：

```
speaker_0002  → 一键训练 → 等待
speaker_0003  → 一键训练 → 等待
...
speaker_0020  → 一键训练 → 等待
```

**总时间估算**：
- GPU: 15-30 小时（可以分批进行）
- CPU: 60-80 小时

---

## 💡 提高效率的技巧

### 技巧 1: 同时训练多个

如果你有多个 GPU，可以同时开多个浏览器窗口训练不同的 speaker。

### 技巧 2: 分批训练

不用一次训练完 20 个，可以：
- 今天训练 5 个
- 明天训练 5 个
- ...

### 技巧 3: 先测试

先训练 speaker_0001，检查效果：
1. 训练完成后，切换到"推理"标签
2. 上传一个测试音频
3. 选择模型：`logs/speaker_0001/G_200.pth`
4. 点击"转换"
5. 听效果

如果满意，继续训练其他。如果不满意，调整参数。

## ✅ 训练完成后的操作

### 1. 整理模型

创建统一存放目录：
```bash
mkdir W:\rvc\trained_models
```

复制模型（以 speaker_0001 为例）：
```bash
mkdir W:\rvc\trained_models\speaker_0001
copy W:\rvc\logs\speaker_0001\G_200.pth W:\rvc\trained_models\speaker_0001\
copy W:\rvc\logs\speaker_0001\added_200.index W:\rvc\trained_models\speaker_0001\
```

### 2. 测试模型

在 RVC WebUI 的"推理"标签页测试。

### 3. 使用推荐系统

```bash
cd W:\fish-speech-1.5.1\voice_factory
C:\Users\moqi\AppData\Local\Programs\Python\Python312\python.exe recommend_voice.py recording.wav
```

## 🎉 总结

### 你需要做的（最少操作）

1. ✅ 双击 `W:\rvc\go-web.bat`（启动 RVC）
2. ✅ 浏览器打开 http://localhost:7897
3. ✅ 点击"训练"标签
4. ✅ 输入 `speaker_0001`，点击"一键训练"
5. ✅ 等待完成

### 我帮你完成的（全自动）

✅ 准备 20 个 speaker 的训练数据（3,250 个音频）  
✅ 质量筛选和 DSP 增强  
✅ 声纹数据库建立  
✅ 推荐系统部署  
✅ 训练文档编写  

### 为什么不能完全自动？

RVC 的训练函数 `train1key()` 是 Gradio WebUI 的生成器函数，需要：
- WebUI 的事件循环
- 浏览器的实时反馈
- GPU 资源的动态分配

这些都是 WebUI 框架的一部分，无法简单剥离。

## 📞 如果遇到问题

### 问题 1: 训练很慢

- 使用 GPU（推荐 RTX 3060 以上）
- 减少 Epoch 到 150
- 减少 Batch Size 到 4

### 问题 2: 显存不足

- Batch Size: 8 → 4 → 2
- 关闭其他程序
- 使用 `--fp16` 选项

### 问题 3: 训练失败

- 检查音频文件格式（必须是 WAV）
- 检查音频时长（至少 2 秒）
- 查看控制台错误信息
